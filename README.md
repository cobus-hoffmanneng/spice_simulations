# PKA Simulations

Git-backed, portable home for the team's **own** SPICE simulation sources — the sim-side sibling
of the Altium `C:\Altium\Libraries` → `altium_lib` pattern.

- **Local path (convention):** `C:\Simulations\` — username-free and **off OneDrive** (like
  `C:\Altium\`), so it's portable across the personal and work machines and heavy generated
  outputs never sync.
- **Remote:** https://github.com/cobus-hoffmanneng/spice_simulations (private — house IP).
- **Owner/users:** Moss owns the plumbing; **Nikola** authors and runs the sims.

## Layout

```
C:\Simulations\
├─ models/            shared SPICE subcircuits/models (.lib .sub .mod .asy) — tracked
├─ scripts/           spicelib runner + smoke test
│  ├─ run_sim.py      portable runner: .cir -> LTspice headless -> parse .raw -> numbers
│  └─ _selftest.cir   RC low-pass smoke test (source)
├─ projects/
│  └─ ELT-HYP/        Eltwin soft-starter sub-circuit sims (Nikola populates)
├─ requirements.txt   pinned spicelib/numpy/matplotlib
├─ .gitignore .gitattributes
└─ README.md
```

New project folders are created **lazily** (only when a real project needs one), like the vault.

## Source vs generated — the one rule to remember

- **SOURCE (tracked):** hand-written netlists are **`*.cir`**. Also `.lib/.sub/.mod/.asy/.asc`
  models & schematics, and everything in `scripts/`.
- **GENERATED (ignored):** `*.raw`, `*.log`, `*.op.raw`, `simout/`, and LTspice's auto-`*.net`
  written beside an `.asc`. If it can be regenerated by re-running the sim, it is not committed.

> Write your decks as `.cir`. Never rely on a `.net` being tracked — it's treated as generated.

## Toolchain — rebuild on a new machine (e.g. the work computer)

The Python venv is **NOT in this repo** (machine-specific). Rebuild it from `requirements.txt`:

```powershell
# 1. Confirm LTspice is installed (NOT pip-installable — get it from analog.com if missing):
Test-Path "$env:LOCALAPPDATA\Programs\ADI\LTspice\LTspice.exe"

# 2. Clone this repo to the standard path:
git clone https://github.com/cobus-hoffmanneng/spice_simulations.git C:\Simulations

# 3. Create the dedicated venv (off-repo, off-OneDrive) and install the pinned toolchain:
python -m venv "C:\Tools\spicelib-venv"
& "C:\Tools\spicelib-venv\Scripts\python.exe" -m pip install -r C:\Simulations\requirements.txt
```

`spicelib` auto-detects the LTspice exe from `%LOCALAPPDATA%`, so nothing here hard-codes a
username or exe path — the same clone works on `CobusHoffmann` and `Cobus`.

## Run a sim

```powershell
# Smoke test (RC low-pass; expects V(out)~0.6321 V at t=1ms):
& "C:\Tools\spicelib-venv\Scripts\python.exe" C:\Simulations\scripts\run_sim.py

# A real deck:
& "C:\Tools\spicelib-venv\Scripts\python.exe" C:\Simulations\scripts\run_sim.py C:\Simulations\projects\ELT-HYP\your_deck.cir
```
